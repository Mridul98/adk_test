## Table Name: REPORTING_DEV.DBT_MAHMUDNABI_gtm_analytics.user_role_snapshot

## Overview

## Logic Explanation  
The query performs a straightforward `SELECT *` from the `sfdc_user_role_snapshot` table, retrieving all columns without any filtering, joins, or aggregations. It simply returns the current state of each role snapshot record, including metadata for change tracking and access permissions.

## Grain of Data  
The result set is at the individual role snapshot record level; each row represents a single snapshot of a user role.
